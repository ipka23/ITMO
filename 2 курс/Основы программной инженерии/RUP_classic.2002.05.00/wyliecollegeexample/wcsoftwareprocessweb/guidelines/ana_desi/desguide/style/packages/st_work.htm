Naming Standard 

The general package naming standards apply to <<work>> packages (See Standards: Package Overview).

Diagramming Standards 

These are ad-hoc work in progress packages – there are no standards for documenting work packages. There should be no <<work>> packages in a build or release of a model.  They are purely transient packages containing work yet to be officially integrated into the model.

Constraints 

None
