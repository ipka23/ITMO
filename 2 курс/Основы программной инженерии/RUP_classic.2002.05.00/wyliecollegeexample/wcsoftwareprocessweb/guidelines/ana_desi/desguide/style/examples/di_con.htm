Standards: Connections Diagram


  Class Diagram 		 The Connections Diagram shows the direct connections between the classes, and interfaces, of a package and the classes, and interfaces, of the packages upon which it is dependent.

Background 

A class diagram showing the other classes (and interfaces)  the classes (and interfaces) in the package depend upon:

  • this diagram is not required if the package does not depend on any other packages
  • this diagram is often combined with the Dependencies Diagram (See Standards: Dependencies Diagram) to produce a single overview showing both the package and class / interface dependencies (See Standards: Dependencies / Connections Diagram).
