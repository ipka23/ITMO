The Design Guidelines describe how to model applications, and components, using the UML and the Rational Rose modeling tool.

The Design Modeling Guidelines are structured around the following concerns:

  1. Model Elements / Applicability

    2. This section outlines which of the standard UML/RUP artifacts are in scope and identifies any extensions, or new artifacts, which are required to support the analysis and design modeling of application's and components.

    It also describes when it is applicable to use each of the in scope artifacts.

    To view the Model Elements / Applicability section click here.

  2. Style Guide / Modeling Standards

    3. This section provides a style guide for each in scope artifact.

    This includes naming and document standards for each in scope artifacts.

    This section is currently under construction.  To view the Standards section click here.

  3. Mappings / Traceability

    4. This sections describes how the analysis and design odel artifacts, and model elements, are derived and / or mapped to other artifacts and model elements from the other models.

    This section is limited to detailing the relationship between the analysis and design model elements and the model elements that they derive from. It does not consider any additional model elements that may be derived from the analysis and design model elements.

    To view the Mappings / Traceability section click here.

  4. Mechanisms

    5. This section documents the mechanisms (strategies or patterns that constitute a common solution to a common problem) that have been developed to simplify and facilitate the analysis and design modeling of applications and components.

    Currently no mechanisms have been discovered / defined for use in the analysis and design models.

    To view the Mechanisms section click here.

  5. Best Practices

This section documents the best practices that have developed whilst applying the design guidelines to the analysis and design modeling of applications and components.

Currently no best practices have been discovered / defined for use in the analysis and design models.
