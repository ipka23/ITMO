Customizing Purify messages

You can change the content and appearance of Purify messages, and control how Purify batches messages.

You can control the content of messages by defining the number of stack frames in the call chain of a message, and specifying whether Purify prints full pathnames in the call chain. You can also decide whether to display various messages such as memory leaks, memory in use, and file descriptors in use when your program exits.
