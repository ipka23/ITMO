Purify overhead

Swap space

At build time, an instrumented program uses virtual memory (real memory plus swap space) equal to two to ten times the size of the program's largest uninstrumented object file or library.

At run time, an instrumented program uses virtual memory equal to about 1.5 times that required by the non-instrumented program. Purify also uses swap space for an error-processing process it creates.

On Tru64 UNIX

At build time, an instrumented program on a Tru64 UNIX platform uses virtual memory considerably larger than the size of the linked executable and shared libraries. Expect to use several hundred megabytes of virtual memory for an average sized application, and up to 600 or 700 MB for larger applications. A minimum of 512 MB of physical RAM and at least 2 GB of swap space is recommended.

Performance

On average, your instrumented program will run two to five times slower than your non-instrumented program. The exact speed depends on how many errors Purify finds in your program, how your program uses memory, and the size of your program's virtual memory relative to your machine's real memory (RAM).

If your instrumented program runs more than five times slower, it might be thrashing, that is, spending an excessive amount of time paging to disk. Use the vmstat command (Solaris, HP-UX, Tru64 UNIX) or osview or gr_osview command (IRIX) to see how much time your program is spending in kernel mode as opposed to user mode.

Typically, a program should be in user mode more than 80 percent of the time. If user mode drops to less than 50 percent, run your program on a machine with more real memory, try to increase the locality of your program's memory references to reduce paging, or increase the real memory of your machine. You can also use the -chain-length option to reduce the call chain length that Purify uses. This in turn reduces Purify's own memory requirements.

Note:

  • The overhead when using Purify and PureCoverage is less than the combined overhead for using these products separately.
