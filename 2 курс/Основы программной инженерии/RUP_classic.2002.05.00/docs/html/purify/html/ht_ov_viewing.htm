Analyzing Purify error messages

The Purify Viewer displays the results of the run of your instrumented program.

  1. Click to expand messages (such as ABR in this example).

  2. For an explanation of a message (such as ABR), look it up in the online Help index.

  3. When you exit the program, analyze the three summary messages in the Viewer: the file descriptors message, the memory leaked summary, and the exit status summary.

The next step in using Purify is to correct errors and rerun the program.
 

What else can you do?

  • Save error data.

  • Customize message appearance, including the number of lines displayed. (By default, it's seven lines.)

  • Submit defects to Rational ClearQuest

Note:

  • If line numbers for error messages are not displayed in the Viewer, make sure you are compiling with the -g option to collect debug data.
